# ה. ניהול ארנונה — מידע מלא לכל נכס

**עדיפות:** בינונית  
**מורכבות:** נמוכה  
**מטרה:** לשמור לכל נכס את כל פרטי הארנונה — מזהים, עלויות ותיעוד

---

## Backend

- [ ] **E-BE-1** — הוסף שדות ארנונה למודל `Property` בסכמת Prisma:
  - `arnonaPayerNumber` — מספר המשלם בעיריה
  - `arnonaPropertyNumber` — מספר הנכס בעיריה
  - `arnonaFileNumber` — מספר תיק ארנונה
  - `arnonaAnnualCost` — עלות ארנונה עדכנית (Decimal)
  - `arnona BillingPeriod` — תקופת חיוב (MONTHLY | BIMONTHLY | ANNUAL)
  - `arnonaCostUpdatedAt` — תאריך עדכון אחרון של הסכום (DateTime)

- [ ] **E-BE-2** — כתוב migration ב-Prisma לשדות החדשים

- [ ] **E-BE-3** — עדכן את ה-DTO של עדכון נכס (`UpdatePropertyDto`) לכלול את שדות הארנונה

- [ ] **E-BE-4** — וודא שה-endpoint `PATCH /properties/:id` מקבל ומחזיר את שדות הארנונה החדשים

---

## Frontend

- [ ] **E-FE-1** — בדף הנכס — הוסף סקשן "פרטי ארנונה" (Accordion / Card נפרד)

- [ ] **E-FE-2** — הצג ועדכן את השדות הבאים בטופס:
  - מספר המשלם (חשוב — tooltip: "מספר זה נדרש לכל פנייה לעיריה")
  - מספר הנכס בעיריה
  - מספר תיק ארנונה
  - עלות ארנונה עדכנית (₪)
  - תקופת חיוב (dropdown: חודשי / דו-חודשי / שנתי)
  - תאריך עדכון אחרון (DatePicker)

- [ ] **E-FE-3** — הצג אינדיקציה "⚠️ המחיר לא עודכן מעל שנה" אם `arnonaCostUpdatedAt` ישן מ-12 חודשים

- [ ] **E-FE-4** — הוסף אפשרות להעלות קובץ שוברי ארנונה (PDF / תמונה) ולצפות בו

- [ ] **E-FE-5** — הצג עלות הארנונה בדוח הוצאות השוטפות של הנכס (חישוב אוטומטי לפי תקופת חיוב)
